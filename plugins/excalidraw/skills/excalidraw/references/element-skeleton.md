# Excalidraw Element Skeleton

Reference for the `ExcalidrawElementSkeleton` format — the JSON schema used when creating elements programmatically via `create_view`.

Source: https://docs.excalidraw.com/docs/@excalidraw/excalidraw/api/excalidraw-element-skeleton

---

## Common Style Properties

Available on all element types (all optional):

| Property | Type | Values |
|---|---|---|
| `width` | number | px |
| `height` | number | px |
| `backgroundColor` | string | hex color |
| `strokeColor` | string | hex color |
| `strokeWidth` | number | `1`, `2`, `4` |
| `strokeStyle` | string | `"solid"`, `"dashed"`, `"dotted"` |
| `fillStyle` | string | `"solid"`, `"hachure"`, `"cross-hatch"`, `"zigzag"` |
| `roughness` | number | `0` (smooth), `1` (default), `2` (rough) |
| `opacity` | number | `0`–`100` |
| `roundness` | object or null | `{ type: 3 }` for rounded corners, `null` for sharp |

## Rectangle, Ellipse, Diamond

**Required:** `type`, `x`, `y`

```json
{ "type": "rectangle", "x": 100, "y": 250 }
{ "type": "ellipse",   "x": 250, "y": 250 }
{ "type": "diamond",   "x": 380, "y": 250 }
```

**With styles:**
```json
{
  "type": "rectangle",
  "x": 50, "y": 250,
  "width": 200, "height": 100,
  "backgroundColor": "#c0eb75",
  "strokeWidth": 2
}
```

## Text Element

**Required:** `type`, `x`, `y`, `text`

```json
{
  "type": "text",
  "x": 100, "y": 100,
  "text": "Hello World"
}
```

**With styles:**
```json
{
  "type": "text",
  "x": 100, "y": 150,
  "text": "Styled Text",
  "fontSize": 20,
  "strokeColor": "#5f3dc4"
}
```

## Text Containers (Shape + Label)

Add a `label` property to any shape instead of using a separate text element. **Preferred over separate text elements.**

**Required:** `type`, `x`, `y`, `label.text`

```json
{
  "type": "rectangle",
  "x": 300, "y": 290,
  "label": { "text": "My Label" }
}
```

**With label styles:**
```json
{
  "type": "diamond",
  "x": -120, "y": 100,
  "width": 270,
  "backgroundColor": "#fff3bf",
  "strokeWidth": 2,
  "label": {
    "text": "Styled Label",
    "strokeColor": "#099268",
    "fontSize": 20,
    "textAlign": "left",
    "verticalAlign": "top"
  }
}
```

Label style properties: `text`, `fontSize`, `strokeColor`, `textAlign` (`"left"`, `"center"`, `"right"`), `verticalAlign` (`"top"`, `"middle"`)

## Lines and Arrows

**Required:** `type`, `x`, `y`

```json
{ "type": "arrow", "x": 100, "y": 20 }
{ "type": "line",  "x": 100, "y": 60 }
```

**With styles:**
```json
{
  "type": "arrow",
  "x": 450, "y": 20,
  "startArrowhead": "circle",
  "endArrowhead": "triangle",
  "strokeColor": "#1971c2",
  "strokeWidth": 2
}
```

Arrowhead values: `"arrow"`, `"triangle"`, `"circle"`, `"bar"`, `null` (no arrowhead)

## Labelled Arrows

```json
{
  "type": "arrow",
  "x": 100, "y": 100,
  "label": { "text": "Labelled Arrow" }
}
```

## Arrow Bindings

Bind an arrow to shapes using `start` and `end`. Pass either `type` (creates a new shape) or `id` (binds to existing element by id).

**Bind by type** (auto-creates and positions shapes):
```json
{
  "type": "arrow",
  "x": 255, "y": 239,
  "label": { "text": "connects" },
  "start": { "type": "rectangle" },
  "end":   { "type": "ellipse" }
}
```

**Bind by id** (connect existing elements):
```json
[
  { "type": "ellipse",  "id": "node-a", "x": 390, "y": 356, "width": 150, "height": 150 },
  { "type": "diamond",  "id": "node-b", "x": -30,  "y": 380, "width": 100 },
  {
    "type": "arrow",
    "x": 100, "y": 440,
    "start": { "id": "node-b" },
    "end":   { "id": "node-a" }
  }
]
```

`start`/`end` optional properties: `type`, `id`, `x`, `y`, `width`, `height`

## Frames

Group elements visually in a named frame.

**Required:** `type`, `children` (list of element ids)

```json
[
  { "type": "rectangle", "id": "r1", "x": 10,  "y": 10 },
  { "type": "diamond",   "id": "d1", "x": 120, "y": 20 },
  { "type": "frame", "children": ["r1", "d1"], "name": "My Frame" }
]
```

---

## Notes

- `id` is regenerated by default — set `regenerateIds: false` if you need stable ids for binding
- If container dimensions are omitted, they are calculated from the label size
- `label` on shapes is preferred over separate `text` elements — fewer elements, automatic centering
- For arrow bindings by `type`, position is computed from the arrow's `x`/`y` when not specified
